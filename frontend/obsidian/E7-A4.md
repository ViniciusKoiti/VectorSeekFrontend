# E7-A4 · Gerar comandos operacionais com contexto

## Descrição técnica
- Criar `CommandGeneratorComponent` que consome `/api/setup/commands` retornando lista de comandos shell com descrição e pré-requisitos.
- Exibir comandos com destaque de sintaxe (`PrismJS`) e botão "Copiar".
- Permitir baixar script agregado (`setup.sh`) contendo comandos selecionados com cabeçalho customizado (logo, instruções).
- Integrar com branding para exibir logotipo e nome da empresa no cabeçalho do script/exportação.
- Adicionar testes assegurando que comandos maliciosos (com `rm -rf`) sejam marcados com aviso.

## Dependências
- [[E7-A1 Implementar wizard de setup operacional]].

## Critérios de aceite
- Usuário visualiza comandos sugeridos com descrição e pré-requisitos claros.
- Exportação gera script shell formatado com logotipo configurável.
- Sistema alerta sobre comandos potencialmente perigosos antes da cópia/execução.

## Checklist sugerido
- [ ] Criar componente de geração de comandos.
- [ ] Integrar endpoint `/api/setup/commands` com validações.
- [ ] Implementar destaque de sintaxe e cópia.
- [ ] Adicionar exportação de script com branding.
- [ ] Escrever testes para validação de comandos suspeitos.
