# E1-A5 · Parametrizar branding e logos

## Descrição técnica
- Criar provider `BRANDING_CONFIG` em `libs/ui/src/lib/branding` expondo informações de logos, cores e mensagens institucionais.
- Permitir troca de logotipo, favicon e cores principais via arquivo JSON (`branding.config.json`) carregado em tempo de build (`fileReplacements`) e variáveis de ambiente.
- Integrar logos configuráveis ao `AppShellLayoutComponent`, telas de autenticação e páginas públicas.
- Garantir fallback seguro quando arquivos não existirem, usando assets padrão.
- Documentar procedimento no README do projeto para personalização white-label.

## Dependências
- [[E1-A3 Construir app shell responsivo]].

## Critérios de aceite
- Alterar `branding.config.json` atualiza logos e cores sem modificar código-fonte.
- Build SSR carrega configurações sem erros em ambientes server/client.
- Storybook demonstra variações com `controls` permitindo trocar URLs de logo.
