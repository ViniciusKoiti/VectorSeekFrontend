# E6-A2 · Implementar página de detalhes de modelo

## Descrição técnica
- Criar rota `/app/models/:id` com `ModelDetailPageComponent` carregando dados via `ModelCatalogService.getModelById`.
- Exibir metadata completa (capabilities, limites, provedor, regiões, status de disponibilidade) com componentes reutilizáveis.
- Tratar erros 403/404/503 exibindo estados dedicados (`AccessDeniedStateComponent`, `ModelUnavailableStateComponent`).
- Integrar seleção de modelo preferido com store (`ModelPreferencesStore`) persistindo escolha.
- Adicionar seção de roadmap mostrando planos de ativação futuros.
