# E1-A1-1 · Provisionar módulo de autenticação

## Objetivo
Estabelecer a estrutura base do módulo `auth` no aplicativo Angular, garantindo roteamento dedicado e isolamento via componentes standalone conforme definido no ADR-001.

## Escopo
- Gerar `apps/platform/src/app/auth/auth.routes.ts` com rotas para login, cadastro e recuperação de senha usando `loadComponent`.
- Criar componentes standalone `LoginPageComponent`, `RegisterPageComponent` e `ForgotPasswordComponent` com templates iniciais e hooks de ciclo de vida configurados.
- Ajustar `app.routes.ts` para lazy loading do módulo de autenticação e aplicação de layout público.
- Configurar `AuthLayoutComponent` (placeholder) para uso futuro, garantindo compatibilidade com SSR.

## Critérios de aceite
- Rotas `/auth/login`, `/auth/register` e `/auth/forgot-password` acessíveis em ambiente local e renderizando componentes placeholders.
- Estrutura do módulo utiliza apenas componentes standalone e `provideRouter` sem `NgModule` auxiliar.
- Lazy loading funcionando sem erros no console (SSR e navegador) usando `ng serve` e `ng run platform:serve-ssr`.

## Evidências esperadas
- Prints ou gravações no PR mostrando navegação entre as rotas públicas.
- Trecho do `app.routes.ts` evidenciando a configuração de `loadChildren` ou `loadComponent` para o módulo `auth`.
- Referência cruzada ao ADR-001 no corpo do PR ou descrição da atividade.
