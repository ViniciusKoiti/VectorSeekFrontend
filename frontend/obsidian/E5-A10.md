# E5-A10 · Testes end-to-end (Cypress ou Playwright)

## Descrição técnica
- Configurar suíte E2E no workspace Nx (`nx g @nx/cypress:project chat-e2e` ou `@nx/playwright`).
- Criar fixtures/mocks para endpoints de chat (`/api/chat/conversations`, `/messages`, `/summarize`, `/models`).
- Implementar cenários: criação de conversa, envio/recepção de mensagem, bloqueio 429, resumo multimídia e seleção de modelo.
- Integrar intercepts para simular streaming (chunks) e erros.
- Publicar comandos de execução (`nx e2e chat-e2e`) no README do frontend e configurar pipeline CI.

## Dependências
- Componentes principais (`E5-A3` a `E5-A8`) implementados para validar fluxo completo.

## Critérios de aceite
- Testes executam com sucesso em CI e localmente, com relatórios de vídeo/screenshot.
- Casos cobrem principais fluxos felizes e falhas (429, erro de resumo).
- Documentação orienta como atualizar mocks conforme backend evolui.
